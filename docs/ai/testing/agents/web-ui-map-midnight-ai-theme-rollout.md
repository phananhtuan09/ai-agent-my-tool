# Web UI Map: midnight-ai-theme-rollout

## Route Inventory
- **Global shell (`base.html`)** ‚Äì fixed 260px sidebar (`aside.sidebar`) plus main stage (`.main-stage`). Includes nav links (`.nav-link`) with numbered icons, the `Menu` button (`button[data-sidebar-open]`), overlay (`.shell-overlay`), topbar actions, and the `#connection-status` badge that reflects `data-state` values (`connecting`, `connected`, `disconnected`). Modal targets mount into `#modal-root`.
- **Dashboard (`/`)** ‚Äì hero banner `.page-hero.accent-blue` followed by `.dashboard-grid` of `.dashboard-card` elements, each with accent strip, status pill, metric list, and `Open ‚Ä¶` action. Hero badge text (Midnight AI / shared runtime) and card copy follow the documented typography tokens.
- **Config (`/config`)** ‚Äì shared runtime hero plus `.config-form`, `.modal-actions` buttons (`Test API`, `Fetch models`, `Save settings`), and the side panel (`aside.panel.side-panel`) with the `.tag-list` of available models or the `.empty-state` fallback. Status pill indicates API key presence.
- **Daily scheduler (`/agents/daily_scheduler`)** ‚Äì accent hero with `agent-model` data roles, chat panel (`#daily-schedule-chat-panel`) hosting the transcript list and form, `#daily-schedule-timeline` data table, and the controls panel inside the collapsible `details` element (`#daily-schedule-controls`). Status pills show planner readiness, while chat messages toggle between `.is-assistant` and `.is-user` list items.
- **Crypto airdrop (`/agents/crypto_airdrop`)** ‚Äì accent hero, filter toggle (`button.filter-toggle`), sidebar (`#airdrop-sidebar`) with controls form and chat panel, and the results panel (`#airdrop-results-panel`) that renders the summary strip, warning stack, and `#airdrop-table` or empty state. Control buttons include `Run crawl now` plus the settings form (`#crypto-airdrop-controls`).

## Screen & State Inventory

- Topbar (`.topbar`) holds `p.eyebrow`, `h1`, `Enable notifications` ghost button, connection badge (`#connection-status`) with `.conn-dot`, and (on agent routes) the `Change model` primary button. ### Dashboard (`/`)

## Selector Strategy
- **Sidebar & navigation** ‚Äì reference `aside.sidebar`, `.nav-link` text (e.g., span text ‚ÄúOverview‚Äù, ‚ÄúConfig‚Äù, agent titles), `.nav-icon` numbering, and `.is-active`. The desktop drawer can be validated by checking for `.sidebar` visibility. For mobile, toggle with `button.nav-toggle[data-sidebar-open]`, close with `button.sidebar-close[data-sidebar-close]`, and blur overlay with `.shell-overlay` (matching `aria-label="Close navigation overlay"`).
- **Global topbar** ‚Äì use `.topbar` to verify the eyebrow/h1 combo, buttons with `btn btn-ghost`, and the connection badge `#connection-status` plus `.conn-dot`. On agent routes, ensure `button.btn.btn-primary` (Change model) is present and `data-state` transitions to ‚Äúconnected‚Äù once SSE updates arrive.
- **Hero + dashboard sections** ‚Äì rely on `.page-hero`, `.hero-badge`, and `.dashboard-card` for layout. For status pills, prefer `.status-pill.is-ready`/`.status-pill.is-warn` classes rather than relying on color, and assert the `dl.metric-list` pairs contain the expected label text (Model/Base URL/API key/Storage).
- **Forms & actions** ‚Äì target `#openai-config-form`, `#daily-schedule-chat-panel form`, `#daily-schedule-controls form`, and `#crypto-airdrop-controls form` by ID and `hx-post` value for stability. Buttons use semantic class combos (`btn btn-primary`, etc.), and `data-loading-text` attributes give deterministic visible text changes.
- **Dynamic fragments** ‚Äì span targets such as `#daily-schedule-chat-panel`, `#daily-schedule-timeline`, `#airdrop-results-panel`, and `#crypto-airdrop-chat-panel` should be captured with their HTMX swap behavior. Use list items `ul.activity-feed.schedule-chat-feed > li` to assert message roles, and table selectors like `#schedule-table`/`#airdrop-table` plus `tbody tr` for rows, leveraging `data-sort`/`tabindex` attributes.

## Interaction Notes
- **HTMX triggers** ‚Äì forms target `/agents/daily_scheduler/chat`, `/agents/daily_scheduler/settings`, `/agents/crypto_airdrop/settings`, and `/agents/crypto_airdrop/chat`. Each swaps its panel via `hx-target`/`hx-swap`, so manual verification should rerun the action before capturing the refreshed fragment. The timeline and airdrop results receive `hx-swap-oob` updates, so include timeline/table refresh in test scripts.
- **Buttons requiring state coverage** ‚Äì the dashboard card CTA, config modal actions, scheduler chat send, schedule settings save, airdrop run, and filter apply buttons all set `data-loading-text`. Capture the default label and the loading label where the design requires animation.
- **Sidebar/mobile drawer** ‚Äì the responsive nav uses `button.nav-toggle` to open and `button.sidebar-close` or the `.shell-overlay` to close. The filter sidebar is hidden/shown by `button.filter-toggle[data-filter-toggle="#airdrop-sidebar"]` and should include the same padding/gutters when toggled on tablet vs desktop.
- **Connection badge** ‚Äì `#connection-status` updates `data-state` (initially `connecting`). Wait for the dot (`.conn-dot`) to enter the `connected` pulse state before taking screenshots that claim successful SSE synchronization.

## Responsive Notes
- **Breakpoints** ‚Äì include desktop (full sidebar + grid), tablet (narrower sidebar, two-column dashboard), and mobile (drawer open/closed) when verifying hero and nav layouts. The configuration sidebar stacks under the main panel on narrower screens, so ensure both the `.panel.page-main` and `.panel.side-panel` remain visible/stacked.
- **Controls & drawers** ‚Äì the crypto filter sidebar and daily controls rely on expandable sections (`details.collapsible-section`), the `filter-toggle` button, and `data-filter-toggle` attributes; capture both expanded and collapsed states. Chron fields pair selects with inputs so keep the label + select + input group visible in responsive screenshots.
- **Button focus & keyboard** ‚Äì tables set `tabindex="0"` and `button` elements include `aria-label` where necessary (sidebar close, overlay). Ensure keyboard focus (via `tabindex`) works on `<tr>` rows and check for accessible text like `aria-controls`/`aria-expanded` on toggles.

## Handoff Summary

### Decisions
- Rely on visible text + semantic classes (`.page-hero`, `.status-pill`, `.dashboard-card`) and IDs (`#connection-status`, `#daily-schedule-chat-panel`) rather than brittle positional CSS. Critical routes are `/`, `/config`, `/agents/daily_scheduler`, and `/agents/crypto_airdrop`, each needing hero + card/form/chat coverage.
- Interactive fragments use HTMX `hx-target` within the same IDs, so tests should capture refreshed states by triggering the associated forms and waiting for the `hx-swap` to complete.

### Blockers
- Need clarity on whether the SSE-driven connection badge always reaches the `connected` pulse state in automated screenshots or if a manual trigger is required.
- The dynamic `available_models` list may be empty in clean state; plan to seed models or capture the empty-state fallback to cover both.

### Open Questions
- Should the verification flow explicitly exercise both scheduled tasks and empty timeline/airdrop states, or is landing on one default state acceptable with notes for the other?
- Is there a preferred way to force a `connected` status on `#connection-status` (SSE stub or manual interaction) when running automated screenshots?
