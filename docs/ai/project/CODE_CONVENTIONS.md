## Naming Conventions

| Context | Convention | Example |
|---------|-----------|---------|
| Variables / functions | `snake_case` | `job_list`, `filter_jobs()` |
| Classes | `PascalCase` | `JobFinderAgent`, `BaseAgent` |
| Constants | `UPPER_SNAKE_CASE` | `DEFAULT_CRON_INTERVAL` |
| Files / modules | `snake_case` | `llm_client.py`, `web_search.py` |
| Pydantic models | `PascalCase` + noun | `JobItem`, `ScheduleTask`, `AirdropResult` |

- Function names must be **verbs**; variables and classes must be **nouns**.
- Be consistent: if `get_jobs()` exists, use `get_airdrops()` not `fetch_airdrops()`.
- No abbreviations except well-known ones (`llm`, `db`, `api`, `url`, `id`).

## Functions / Methods

- **Single responsibility**: one function, one job.
- **Max 2-3 arguments**: if more needed, pass a Pydantic config object.
- **Avoid side effects**: return new values, do not mutate inputs.
- **Async by default**: all FastAPI routes and agent methods use `async def`.

```python
# Good
async def filter_jobs(jobs: list[JobItem], config: JobFilterConfig) -> list[JobItem]:
    ...

# Bad
def filterJobs(jobs, salary_min, salary_max, locations, keywords):
    ...
```

---

## Control Flow

- **Guard clauses first**: handle errors/edge cases at top, main logic last.
- **Max nesting**: 2-3 levels. Extract deeper logic into named functions.
- **No bare `except`**: always catch specific exception types.

```python
# Good
async def run(self) -> AgentResult:
    if not self.config.enabled:
        return AgentResult.disabled()
    if not self.api_key:
        raise ConfigError("API key missing for agent")
    return await self._execute()
```

---

## Error Handling

- Raise with **clear, actionable messages**.
- Never swallow exceptions silently.
- Domain exception hierarchy:
  - `AgentError` → base for all agent failures
  - `ConfigError` → missing/invalid configuration
  - `CrawlError` → Playwright / crawler failures
  - `LLMError` → Anthropic / OpenAI API failures
- FastAPI routes use `HTTPException` with meaningful `detail` strings.

## Python-specific Rules

- **Type hints**: required on all public function signatures and class attributes.
- **Pydantic v2**: use for all request/response bodies, config objects, and data models.
- **f-strings**: preferred over `.format()` or `%`.
- **`pathlib.Path`**: preferred over `os.path`.
- **Async I/O**: no blocking calls inside `async def`. Use `asyncio.to_thread()` if wrapping sync code.
- **`asyncio.gather()`**: for concurrent operations (e.g., crawling multiple job sites in parallel).

## SSE / Streaming Pattern

Agent output streams two event types:

```python
# Chat text token → chat panel
{"type": "chat", "content": "token..."}

# Tool call result → display panel update
{"type": "ui_update", "panel": "schedule", "data": [...]}

# Cron notification
{"type": "notify", "message": "1 hour passed, update progress!"}
```
