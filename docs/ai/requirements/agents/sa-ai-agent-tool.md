## 1. Executive Summary

**Feasibility Verdict**: Feasible with Changes
**Complexity**: High

**Key Recommendations**:
- Run Playwright in `asyncio.to_thread()` — not directly in the FastAPI event loop — to avoid blocking SSE connections during crawl operations.
- Enable SQLite WAL mode on all `memory.db` files to eliminate write contention between APScheduler cron jobs and web request handlers.
- Use a per-agent in-memory asyncio `Queue` as the SSE broadcast channel; decouples cron push events from HTTP response handlers cleanly.
- DuckDuckGo search: jitter-randomized delays (2–5s between requests) + result cache keyed to crawl cycle timestamp.
- LLM hot-swap: store config in `settings.yaml`; `BaseAgent.reload_llm_client()` replaces instance in place — no server restart.
- Confirm airdrop target sites before Phase 4 (OQ-02 is blocking for Crypto Airdrop agent crawler).
- Use asyncio Lock in `shared/crawler.py` to serialize Playwright sessions and prevent OOM when two crawl jobs overlap.

**Major Risks**:
- Playwright inside single Python process can exhaust VPS memory (400–600MB per Chromium). Two overlapping crawl jobs → OOM risk.
- TopCV, ITviec, VietnamWorks actively detect headless browsers; anti-blocking adds complexity and may still fail.
- DuckDuckGo unofficial library has no SLA; may break without notice.
- SSE connections held open by HTMX require careful FastAPI lifespan management; orphaned generators on navigation = memory leaks.

## 3. Recommended Architecture

### Pattern: Layered Monolith with Agent Registry

**Justification**: Single-user personal tool on VPS. Monolith minimizes operational complexity. Agent registry provides extensibility (FR-06) without speculative abstractions. FastAPI asyncio event loop aligns with APScheduler AsyncIOScheduler and Playwright async API.

```
Browser (HTMX)
    │  HTTP GET/POST (partial HTML)
    │  SSE connection (/stream/{agent})
    ▼
FastAPI (main.py)
    │
    ├──▶ api/job_finder.py ──▶ agents/job_finder/agent.py
    │                              ├── tools.py (crawl, filter, rank)
    │                              ├── skills.py (prompts)
    │                              ├── memory.db
    │                              └── cron.py → asyncio Queue
    │
    ├──▶ api/daily_scheduler.py ──▶ agents/daily_scheduler/agent.py
    │                                   ├── tools.py
    │                                   ├── skills.py
    │                                   ├── memory.db
    │                                   └── cron.py → asyncio Queue
    │
    ├──▶ api/crypto_airdrop.py ──▶ agents/crypto_airdrop/agent.py
    │                                  ├── tools.py
    │                                  ├── skills.py
    │                                  ├── memory.db
    │                                  └── cron.py → asyncio Queue
    │
    └──▶ api/stream.py  ←── SSE fan-out per agent Queue
                              ▲
                              ├─ LLM token stream (chat events)
                              ├─ tool call results (ui_update events)
                              └─ cron completion (notify events)

shared/
  ├── llm_client.py   (Anthropic + OpenAI unified wrapper, hot-swap)
  ├── crawler.py      (Playwright, asyncio.to_thread, global Lock)
  └── web_search.py   (duckduckgo-search, rate-limited, cached)
```

## 6. Key Technical Patterns

### LLM Hot-Swap (FR-02, BR-09)
- `settings.yaml` stores per-agent: `provider`, `model`, `api_key_env_var`
- `LLMClient` is an instance attribute on each agent (not module singleton)
- `PUT /agents/{name}/config` rewrites yaml section
- `BaseAgent.reload_llm_client()` creates new `LLMClient` in place; no restart

### Playwright Subprocess Isolation (FR-07, FR-22)
- All Playwright calls wrapped in `asyncio.to_thread()` using sync Playwright API
- Global `asyncio.Lock()` in `shared/crawler.py` — only one browser session at a time
- Prevents OOM when daily job crawl and 6-hour airdrop crawl overlap
- Chromium launched with `--disable-dev-shm-usage` to reduce memory footprint

### SSE Fan-Out Pattern (FR-04, FR-18)
- Each agent has one `asyncio.Queue` instance in memory
- Multiple event producers: LLM token stream, tool call results, cron jobs
- SSE endpoint `/stream/{agent}` consumes from Queue, yields formatted SSE events
- On disconnect: generator catches `GeneratorExit`; removes Queue consumer; prevents leak

## 7. Data Models

```sql
-- agents/job_finder/memory.db
CREATE TABLE jobs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    title       TEXT NOT NULL,
    company     TEXT,
    salary_min  INTEGER,
    salary_max  INTEGER,
    location    TEXT,
    tech_stack  TEXT,       -- JSON array
    source      TEXT,       -- "topcv" | "itviec" | "vietnamworks"
    url         TEXT NOT NULL,
    ai_score    INTEGER,    -- 0-100; NULL before ranking
    ai_reason   TEXT,
    crawled_at  DATETIME NOT NULL
);
CREATE INDEX idx_jobs_crawled ON jobs(crawled_at);
CREATE INDEX idx_jobs_score ON jobs(ai_score DESC);

-- agents/daily_scheduler/memory.db
CREATE TABLE tasks (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    title             TEXT NOT NULL,
    estimated_minutes INTEGER,
    start_time        DATETIME,
    end_time          DATETIME,
    status            TEXT,   -- "pending"|"in_progress"|"done"|"deferred"|"dropped"
    created_at        DATETIME NOT NULL
);

-- agents/crypto_airdrop/memory.db
CREATE TABLE crawl_cycles (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at   DATETIME NOT NULL,
    completed_at DATETIME
);
CREATE TABLE airdrops (
    id                   INTEGER PRIMARY KEY AUTOINCREMENT,
    crawl_cycle_id       INTEGER REFERENCES crawl_cycles(id),
    name                 TEXT NOT NULL,
    chain                TEXT,
    ai_score             INTEGER,   -- 0-100
    requirements_summary TEXT,
    deadline             DATETIME,
    source_url           TEXT,
    crawled_at           DATETIME NOT NULL
);
CREATE INDEX idx_airdrops_cycle ON airdrops(crawl_cycle_id);
CREATE INDEX idx_airdrops_score ON airdrops(ai_score DESC);
```

## 8. Technical Edge Cases

| ID | Category | Edge Case | Expected Behavior | Priority |
|----|----------|-----------|-------------------|----------|
| TE-01 | Concurrency | Two crawl jobs fire simultaneously | asyncio Lock in crawler.py serializes; second waits; user sees "crawl in progress" notify | Must |
| TE-02 | Concurrency | User sends chat while cron LLM call running for same agent | Per-agent `is_processing` flag; return "Agent is thinking..." immediately if True | Must |
| TE-03 | Concurrency | APScheduler writes memory.db while handler reads | SQLite WAL allows concurrent reads + one writer; no corruption | Must |
| TE-04 | Data | Zero jobs pass hard filter | Show empty state card; skip AI ranking call (saves cost) | Must |
| TE-05 | Data | LLM returns malformed JSON | Parse with try/except; fallback to raw text or error card; no crash | Must |
| TE-06 | Data | Job site returns empty results page | Treat as zero-result crawl (not error); store crawl timestamp; show info message | Should |
| TE-07 | Data | DuckDuckGo returns no results | Log warning; skip web search context; AI analyzes with available data only | Should |
| TE-08 | Data | settings.yaml malformed after edit | Catch yaml.YAMLError; return 400; keep last valid in-memory config unchanged | Must |
| TE-09 | Data | memory.db missing or corrupt on startup | Auto-create via CREATE TABLE IF NOT EXISTS; log warning if corrupt | Must |
| TE-10 | Network | Playwright timeout on job site (>30s) | Raise CrawlError; continue other sources; emit warning SSE notify | Must |
| TE-11 | Network | LLM API timeout or 5xx | Raise LLMError; retry once after 3s; push error to SSE chat stream on second failure | Must |
| TE-12 | Network | SSE connection drops mid-stream | Generator catches GeneratorExit; cleanup Queue listener; no leak | Must |
| TE-13 | Security | Crawled HTML content contains XSS payload | Jinja2 auto-escaping enabled; never use `\| safe` filter on crawled content | Must |
| TE-14 | Security | API key stored in settings.yaml | chmod 600 on file; never return key values in API responses or logs | Must |
| TE-15 | Performance | 200+ jobs require AI ranking | Batch into groups of 20; asyncio.gather() per batch; stream partial results as batches complete | Must |
| TE-16 | State | Overdue task interactive handling (FR-20) | Agent checks end_time < now() in LLM context; flags overdue tasks; waits for user reply before rescheduling | Must |
| TE-17 | State | Midnight purge while user is using app | APScheduler fires DELETE; emit SSE notify "Day reset"; timeline re-renders to empty state | Should |
| TE-18 | Integration | DuckDuckGo rate-limited | Exponential backoff with jitter (2–8s); cache per crawl cycle; after 3 retries skip that search | Must |
| TE-19 | Integration | Job site changes HTML structure | CrawlError raised; partial results from working sources shown; selector failure logged | Must |

**Prerequisite**: Manual crawl test on TopCV, ITviec, VietnamWorks with playwright-stealth before coding.

DuckDuckGo wrapper, airdrop site crawlers, AI analysis skill template, 6-hour cron, 10-cycle purge, chat + cards UI.
**Prerequisite**: Confirm airdrop target sites (OQ-02) before starting.

### Phase 5: LLM Config Modal + Polish

Gear icon modal, hot-swap implementation, browser notification permission flow, error state UI, VPS deployment guide.
